## 命名规范

### 卡牌 ID
- 格式：`CHARMOD-<SNAKE_CASE>`
- 示例：`CHARMOD-LETHAL_RHYTHM`
- 类名：`LethalRhythm.cs`（PascalCase）

### 能力 ID
- 格式：`CHARMOD-<SNAKE_CASE>_POWER`
- 示例：`CHARMOD-POISON_POISON_POWER`
- 类名：`PoisonPoisonPower.cs`

### 图片命名规则
引擎通过 `Id.Entry.RemovePrefix().ToLowerInvariant()` 自动查找图片：
- `CHARMOD-POISON_POISON_POWER` → `poison_poison_power.png`
- 路径：`CharMod/images/powers/poison_poison_power.png`
- 大图：`CharMod/images/powers/big/poison_poison_power.png`

```csharp
public sealed class CardName : CharModCard

    protected override IEnumerable<DynamicVar> CanonicalVars =>
        new DynamicVar[]
        {
            new IntVar("Damage", 10m),
            new IntVar("Block", 8m),
        };

    // HoverTip（鼠标悬停提示）
    protected override IEnumerable<IHoverTip> ExtraHoverTips =>

    {
    }

    // 打出效果
    protected override async Task OnPlay(PlayerChoiceContext choiceContext, CardPlay cardPlay)
    {

            .FromCard(this, cardPlay)
            .Targeting(choiceContext.SelectedCreature)
            .WithHitFx("vfx/vfx_attack_slash")
            .Execute(choiceContext);
    }

        base.Damage.UpgradeBy(4);

    }
```

## 多人游戏同步（⚠️ 最重要）

### 问题根源

多人游戏中，Power 实例在克隆时会被重置，**实例字段不会保留状态**，导致两端不同步。

### 解决方案：使用 `InternalData`

```csharp
// ❌ 错误：实例字段（多人游戏不同步）
private int _counter = 0;
private bool _isApplying = false;

// ✅ 正确：InternalData（多人游戏同步）
private class Data
{
    public int counter;
    public bool isApplying;
}

protected override object InitInternalData()
{
    return new Data();
}

// 使用时：
var data = GetInternalData<Data>();
data.counter++;
```

### 集合遍历安全

```csharp
// ❌ 错误：直接遍历可能在循环中修改集合
foreach (var enemy in base.CombatState.Enemies)

// ✅ 正确：过滤+ToList 创建副本
foreach (var enemy in base.CombatState.Enemies.Where(e => e.IsAlive).ToList())
```

### 全局永久增加（参考遗传算法）

```csharp
// 使用 [SavedProperty] 实现跨战斗永久增加
[SavedProperty]
public int GlobalDamageBonus { get; set; }

// 在 DeckVersion 中存储
public override void OnDeckVersionCreated()
{
    GlobalDamageBonus = 0;
}
```

## 常见坑

### 1. 文本描述和代码不一致
- **现象**：卡牌描述说"全体"，代码只打单体
- **解决**：每次修改代码后同步更新文本

### 2. 多人游戏不同步
- **现象**：联机时两端状态不一致
- **解决**：所有状态用 `InternalData`，不用实例字段

### 3. 图片不显示
- **现象**：能力图标显示为默认图片
- **解决**：检查命名是否符合 `id.ToLowerInvariant()` 规则

### 4. 卡牌费用/数值写错
- **现象**：伤害139这种明显bug
- **解决**：仔细检查构造函数和 `OnUpgrade`

### 5. 本地化文件不同步
- **现象**：zh 改了但 eng 没改
- **解决**：同时更新两个目录的 json 文件

接下来我们详细解释一下每个属性的作用：

Title 属性表示此卡池的标题，这个标题名称会影响卡面图资源路径。
还记得自定义卡牌那一期专栏中，卡面图(肖像图)默认从 res://images/atlases/card_atlas.sprites/<卡池名称>/<卡牌ID的小写表示>.tres 这类路径找。

而 Title 属性就表示这个卡池的名称。

EnergyColorName 属性表示卡池卡牌能量图标名称，也就是卡牌左上角的能量图标资源名称。

游戏本体会寻找位于 res://images/atlases/ui_atlas.sprites/card/energy_<EnergyColorName>.tres 的资源作为卡牌的能量图标。


CardFrameMaterialPath 属性表示卡牌边框的着色器材质资源名称，游戏会在：

res://materials/cards/frames/<CardFrameMaterialPath>_mat.tres 这个路径去加载卡牌边框的着色器材质，自然，它会影响整个卡面的颜色以及效果。

DeckEntryCardColor 属性控制卡池中的卡的小卡片/牌堆列表那种缩略卡的底色，也就是控制打出后卡牌化作缩略卡牌的颜色。

EnergyOutlineColor 属性控制卡牌的卡左上角能量数字/星星数字的描边颜色。

IsColorless 属性表示当前卡池是否为无色卡池。

GenerateAllCards 这个方法用于指定当前自定义卡组中的所有卡牌，返回一个 CardModel 数组，我们演示的代码就拿两张铁血战士的卡牌作为占位符。

shader_type canvas_item;

uniform float h: hint_range(0,1) = 1;
uniform float s: hint_range(0,5) = 1;
uniform float v = 1;

varying vec4 modulate_color;

void vertex() {
    modulate_color = COLOR;
}

void fragment() {
    mat3 RGB_to_YIQ = mat3(
        vec3(0.2989,  0.5959,  0.2115),
        vec3(0.5870, -0.2774, -0.5229),
        vec3(0.1140, -0.3216,  0.3114));

    vec4 col = texture(TEXTURE, UV);

    col.rgb = RGB_to_YIQ * col.rgb;
    float hue = 1.0 - h;
    hue = mix(0, 6.283185, hue);

    float sin_hue = sin(hue);
    float cos_hue = cos(hue);
    mat3 hue_shift = mat3(
        vec3(1.0, 0, 0),
        vec3(0, cos_hue, -sin_hue),
        vec3(0, sin_hue, cos_hue));
    col.rgb *= hue_shift;

    mat3 sat_shift = mat3(
        vec3(1.0, 0, 0),
        vec3(0, s, 0),
        vec3(0, 0, s));
    col.rgb = sat_shift * col.rgb;

    col.rgb = mix(vec3(0,0,0), col.rgb, v);

    col.rgb = inverse(RGB_to_YIQ) * col.rgb;
    COLOR = col;
    COLOR *= modulate_color;
}

检索程序集脚本
杀戮尖塔2在尝试加载模组时，直接使用了 AssemblyLoadContext.LoadFromAssemblyPath 这个方法从模组路径加载.NET程序集的动态链接库文件。

这一步导致了一个大问题，Godot 初始化 .NET 模块时，会检索游戏关联程序集中的类脚本，建立脚本资源的关系链映射，当我们在模组中添加了一个带有脚本的场景后，当游戏在尝试实例化这个场景时，由于杀戮尖塔2在尝试加载模组时绕过了 Godot 内部的检索机制，脚本资源与程序集未能建立有效的映射关系，导致后续会出现找不到程序集的错误。

我们需要调用 Godot 内部检索程序集的方法实现修复，在模组的 Initializer 中的初始化方法中添加以下代码即可。

复制代码
